FealtyX - GoLang/Java/Python REST API Assignment
Overview
This project demonstrates the creation of a simple REST API that manages student data (ID, Name, Age, Email). The API performs basic CRUD (Create, Read, Update, Delete) operations and integrates with the Ollama AI platform to generate summaries of student profiles.

Project Setup
This project uses Go for creating the REST API. Ensure you have Go installed on your system. If not, follow the instructions here to install Go.

Steps to Set Up:
Clone the repository.
Run go mod init <module-name> to initialize the Go module.
Run go run main.go to start the server.
API Endpoints
The API exposes the following endpoints:

1. Create a New Student
Method: POST
Endpoint: /students
Request Body:
json
Copy code
{
    "name": "John Doe",
    "age": 21,
    "email": "john@example.com"
}

Response:
json
Copy code
{
    "id": 1,
    "name": "John Doe",
    "age": 21,
    "email": "john@example.com"
}

Get All Students
Method: GET
Endpoint: /students

Response:
json
Copy code
[
    {
        "id": 1,
        "name": "John Doe",
        "age": 21,
        "email": "john@example.com"
    },
    {
        "id": 2,
        "name": "Jane Smith",

Get a Student by ID

json

    "id": 1,

Update a Student by ID
Method: PUT
Endpoint: /students/{id}
Request Body:
json
Copy code
{

    "age": 22,
    "email": "johnathan@example.com"
}


Delete a Student by ID
Method: DELETE
Endpoint: /students/{id}
Response: Student deleted successfully
6. Generate a Summary of a Student by ID using Ollama
Method: GET
Endpoint: /students/{id}/summary
Response: A summary generated by Ollama AI, describing the student's profile.

Install the Llama3 language model for Ollama.
Make API requests to Ollama from your Go application. Pass the student object as input to generate a summary.

Data Storage
An in-memory data structure (a slice) is used to store student information. This avoids the need for a database for this assignment.

Error Handling
If an invalid ID is requested, the API will return a 404 Not Found response.
If an invalid input is provided (e.g., missing name or invalid age), the API will return a 400 Bad Request.
Input Validation
Before creating or updating a student, the following checks are made:

The name field cannot be empty.
The age field must be a positive integer.
The email field must be a valid email address.
Concurrency
The application is designed to handle concurrent requests safely by using Go’s goroutines and channels.
